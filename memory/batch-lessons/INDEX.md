# batch-lessons — Memory Index

> Procedural lessons `/batch-linear-tickets` learns across batches, about its
> **own orchestration** (ticket-type classification, cross-ticket correlation,
> chronic Needs-Info shapes). Planning / implementation lessons are inherited
> from `aw-lessons` (written by the `aw-planner`/`aw-executor` fan-out), not
> stored here. This is the **fast tier** of the batch self-improvement loop —
> see [`skills/workflow/batch-linear-tickets/rules/self-improvement-loop.md`](../../skills/workflow/batch-linear-tickets/rules/self-improvement-loop.md).
>

> affected-area. Written at Phase 5 (results). Managed by `/persistent-memory`.
>
> Keep ≤ 200 lines. When it exceeds 200, run
> `/persistent-memory consolidate batch-lessons`.
